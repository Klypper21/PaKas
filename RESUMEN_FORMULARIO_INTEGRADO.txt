┌─ ANTES: Flujo Antiguo ─────────────────────────────────────────────────────┐
│                                                                              │
│ Admin Panel                                                                 │
│ ├─ Panel Productos: [Nuevo] → Abre Modal                                  │
│ │  ├─ Form.nombre                                                           │
│ │  ├─ Form.talla (un solo valor)                                            │
│ │  ├─ Form.colores (selector visual)                                        │
│ │  ├─ Form.precio (un solo valor)                                           │
│ │  └─ Form.stock (un solo valor)                                            │
│ │     ↓ Guardar → 1 producto con precio/stock único                        │
│ │                                                                            │
│ └─ Panel Variaciones: (separado!) → Agregar variaciones manualmente       │
│    ├─ Para cada variación:                                                  │
│    │  ├─ Modal de variación                                                 │
│    │  ├─ Ingresa: SKU, Color, Talla, Precio, Stock                         │
│    │  └─ Clic "Guardar variación"                                           │
│    └─ Resultado: Variaciones dispersas en otro panel                       │
│                                                                              │
│ ❌ Problemas:                                                               │
│    • Flujo separado y confuso                                              │
│    • Admin debe ir a 2 paneles diferentes                                  │
│    • Fácil olvidar agregar variaciones                                     │
│    • No hay validación de consistencia                                     │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ FLUJO DE DATOS AL GUARDAR ───────────────────────────────────────────────┐
│                                                                              │
│ 1. Usuario completa formulario y clicks [Guardar]                          │
│    ↓                                                                         │
│ 2. productForm.addEventListener('submit') se ejecuta                       │
│    ↓                                                                         │
│ 3. Obtener variaciones de window.ProductVariationsBuilder                  │
│    Ejemplo: [                                                               │
│      {color: "Rojo", talla: "S", stock: 10, price: 25, sku: "CAMI-RED-S"}│
│      {color: "Rojo", talla: "M", stock: 15, price: 25, sku: "CAMI-RED-M"}│
│      ...                                                                    │
│    ]                                                                        │
│    ↓                                                                         │
│ 4. Calcular precio promedio y stock total de variaciones                   │
│    ↓                                                                         │
│ 5. Construir payload del producto (price y stock ya llenos)                │
│    ↓                                                                         │
│ 6. INSERT or UPDATE en tabla "products"                                    │
│    ↓                                                                         │
│ 7. Si es UPDATE (edición): DELETE antiguas variaciones                     │
│    ↓                                                                         │
│ 8. INSERT nuevas variaciones en tabla "product_variations"                 │
│    ↓                                                                         │
│ 9. Notificación de éxito                                                    │
│    ↓                                                                         │
│ 10. Modal se cierra y lista de productos se recarga                        │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
